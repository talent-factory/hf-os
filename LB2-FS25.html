1. Git-Aufgaben Anleitung

1.1. Lokalen Branch erstellen

Der Branch soll der folgenden Syntax entsprechen: <Klasse>/<Vorname>-<Nachname>

Beispiel:

# Wechsle zum Hauptbranch (ist bereits erfolgt)
git checkout main

# Erstelle einen neuen Branch mit deinem Namen
git checkout -b hfi2024a/max-mustermann

1.2. Branch ins Remote Repository pushen

# Pushe deinen neuen Branch ins Remote Repository
git push -u origin hfi2024a/max-mustermann

1.3. Aufgaben bearbeiten

  • Speichere deine Lösungen in Dateien mit der Namenskonvention aufgabe-X.txt.

  • Beispiel: aufgabe-1.txt

1.4. Änderungen committen und pushen

# Füge alle Änderungen hinzu
git add .

# Committe die Änderungen mit einer aussagekräftigen Nachricht
git commit -m "Lösung für Aufgaben hinzugefügt"

# Pushe die Änderungen in deinen Branch
git push

Wichtige Hinweise

  • Arbeite nur in deinem persönlichen Branch

  • Halte deine Dateien übersichtlich und benenne sie klar

  • Bei Fragen wende dich an deinen Dozierenden

Tipps

  • Committe regelmässig deine Fortschritte

  • Verwende aussagekräftige Commit-Nachrichten

  • Achte auf die korrekte Branch-Benennung

3. Erweiterte Datei Operationen | 10P | 15'

Wir geben auf der Konsole folgende Befehle in genau dieser Reihenfolge ein. Beschreibe jeden einzelnen Schritt/Befehl detailliert und erkläre die zugrundeliegenden Konzepte.

  • Was genau macht jeder dieser Befehle und welche Konzepte des Dateisystems werden dabei angewendet?

    • mkdir -p test/subdir && cd test

    • echo "Inhalt-$(date +%s)" > original.txt

    • ln -s original.txt symlink.txt

    • ln original.txt hardlink.txt

    • chmod 640 original.txt

    • touch -d "2 days ago" original.txt

    • ls -la

  • Kopiere die Ausgabe des letzten Befehles als Anhang zu der Lösung.

  • Analysiere die Ausgabe im Detail:

    • Erkläre die Bedeutung jeder Spalte in der Ausgabe

    • Welche Unterschiede bestehen zwischen den drei Dateien bezüglich Inode, Berechtigungen und Zeitstempel?

    • Warum haben bestimmte Dateien die gleiche Inode-Nummer und andere nicht?

  • Als letzte Befehle geben wir noch folgende Zeilen ein:

    • echo "Neuer Inhalt" > original.txt

    • cat symlink.txt hardlink.txt

    • mv original.txt ../original.txt

    • cat symlink.txt hardlink.txt 2>&1

      Erkläre für jeden Befehl:

    • Was wird ausgegeben und warum?

    • Welche Auswirkungen haben die Befehle auf die Inode-Tabelle?

    • Wie unterscheidet sich das Verhalten von symbolischen und harten Links in diesem Szenario?

    • Welche Fehler könnten auftreten und warum?

  • Bonus-Frage:

    • Wie würde sich das Verhalten ändern, wenn die Dateien auf unterschiedlichen Dateisystemen (Partitionen) liegen würden?
